Credit card fraud detection

Python Random Forest Imbalanced data

Problem

Financial fraud losses exceed $30B annually. Existing models suffer from high false positives and poor recall on rare fraudulent transactions.

Approach

Used Random Forest with SMOTE balancing and feature engineering (transaction velocity, merchant categories). Achieved 0.98 ROC-AUC.

Architecture

Ensemble of 200 trees, hyperparameter tuning via GridSearch, deployed as Flask API with monitoring dashboard.

Implementation

Python, scikit-learn, imbalanced-learn. Jupyter for prototyping, then refactored into production-ready modules.

Outcome

96% recall on fraud cases, 40% reduction in false positives compared to logistic regression. (in‑development)

Lessons

Feature importance revealed that time‑since‑last‑transaction was critical. Learned to handle class imbalance without leaking.

FPGA gesture recognition (PYNQ)

PYNQ-Z1 Vivado HLS CNN

Problem

Real‑time gesture recognition on edge devices requires low latency and low power; CPUs insufficient.

Approach

Quantized CNN implemented on FPGA using PYNQ framework. Used Python overlay for control.

Architecture

3‑layer CNN with batch normalization, mapped to programmable logic via HLS; DMA for data transfer.

Implementation

Vivado HLS for IP, PYNQ for driver. Trained on 10k images, converted to fixed‑point.

Outcome

Latency 8ms, power <2W, accuracy 92%. Achieved 5x speedup over ARM Cortex-A9.

Lessons

Fixed‑point tuning is crucial; pipelining reduced latency further. Open‑source release planned.
